# Tonpalett — Enhancior.se Copy Style Guide

## Ton

Underdrift slår överdrift. Fikarum, inte konferensscen.

- **Direkt, inte dramatisk.** Vi beskriver problem och lösningar rakt — ingen överdriven krisretorik.
- **Teknisk trovärdighet.** Vi låter som en senior arkitekt som pratar med en kollega, inte en säljare som pratar med en prospect.
- **Lugn auktoritet.** Erfarenhet behöver inte skrika. Tonen är trygg, inte brådskande.
- **Konkret framför abstrakt.** Leverabler, tidslinjer, utfall. Inte "holistiska ramverk" eller "transformativa insikter".

## Bannlista

Dessa ord är förbjudna i all copy. Använd ersättningen istället.

| Förbjudet | Ersättning |
|-----------|------------|
| fraktionell | deltid / inbäddad / på deltid |
| exekverbar | konkret / genomförbar |
| intervention | insats / åtgärd |
| impakt | nytta / effekt / påverkan |
| akut (som tjänsteadjektiv) | ta bort eller omformulera |
| blöder / blöda (om system) | dränerar / läcker / bromsar |
| holistisk | helhetsbild / övergripande |
| transformativ | konkret förbättring |
| proaktiv (som säljord) | förebyggande / i förtid |

## Formatregler

- **Inga em-dash (—).** Använd punkt, komma, kolon eller parentes istället.
- **En-dash (–)** OK för intervall (t.ex. "1–2 dagar", "3–12 månader").
- **Inga emojier** i brödtext eller rubriker.
- **Inga utropstecken** i brödtext. OK i CTA-knappar om det verkligen behövs (det gör det sällan).
- **Konsekvent du/ni.** Vi använder "ni" till prospekten (företaget). Aldrig "du" i marknadsföringscopy.

## Metaforvokabulär

Godkända metaforer för att beskriva teknisk skuld:

- **slukål** — system som slukar resurser utan synligt resultat
- **dränera** — sprint-kapacitet som rinner ut
- **läcka** — prestanda/kvalitet som gradvis försämras
- **bränna** — kapital som spenderas på fel prioriteringar
- **bromsa** — teknisk skuld som saktar ner leveransen

Förbjuden metafor: **blöda** (om system — för dramatiskt och AI-klingande).
